# 数据/算法实习生 - 岗位专项评审标准

> 本文件为岗位叠加层，继承 `_base-template.md` 的通用评审框架。
> 以下内容占据总分的 **15%（岗位专项维度）**。

## 岗位专项评审维度

### 1. 数据处理与清洗 (3分)

- 数据源接入是否可靠（异常数据处理、空值填充策略）
- 数据清洗逻辑是否完整（去重、类型转换、异常值处理）
- 数据质量校验是否到位（schema 校验、行数校验、字段完整性）
- 大数据量处理是否考虑性能（分批处理、流式处理、避免全量加载到内存）
- 数据管道是否有幂等性（重跑不会产生脏数据）
- 是否有数据血缘记录（数据从哪来、怎么处理、到哪去）

### 2. 模型与算法 (3分)

- 特征工程是否合理（特征选择、缺失值处理、编码方式）
- 模型选择是否有依据（为什么选这个算法、是否有对比实验）
- 训练/验证/测试集划分是否正确（数据泄漏检查）
- 超参数调优是否有记录（实验追踪、参数搜索日志）
- 模型评估指标是否全面（不只看准确率，还看精确率/召回率/F1/AUC）
- 过拟合/欠拟合是否识别和处理

### 3. 代码质量 (3分)

- 数据处理管道是否模块化（而非一个巨大的脚本）
- 硬编码的路径、阈值、参数是否提取为配置
- 是否有数据版本管理（数据集版本、模型版本）
- 是否有实验记录（MLflow/W&B/自建日志）
- 可复现性（随机种子固定、环境锁定、依赖锁定）
- 注释是否充分（算法选择理由、参数含义、数据假设）

### 4. SQL 与查询 (3分)

- SQL 查询是否高效（避免全表扫描、合理使用分区）
- 窗口函数/CTE 使用是否正确
- 大查询是否有优化（物化视图、预聚合、增量计算）
- 数据口径是否统一（指标定义是否有文档、是否与业务对齐）
- 数据权限是否考虑（敏感字段脱敏、访问控制）

### 5. 可视化与输出 (2分)

- 数据报告是否清晰（图表选择是否合适、标注是否完整）
- 结论是否有数据支撑（不说"感觉"，说"数据显示"）
- 异常波动是否有归因分析
- 输出格式是否便于下游使用（CSV/Parquet/API）

## 数据专项扣分示例（供AI参考）

| 问题 | 扣分 | 理由 |
|------|------|------|
| 训练集数据泄漏到测试集 | -10 | 实验结果不可信 |
| 全量数据加载到内存 | -8 | OOM 风险 |
| 硬编码绝对路径 | -3 | 不可移植 |
| 只看准确率不看其他指标 | -5 | 评估不全面 |
| 无实验记录 | -3 | 不可追溯 |
| SQL 全表扫描无分区 | -5 | 查询性能差 |
| 数据清洗缺失值直接删行 | -3 | 数据损失 |
