# 运维/DevOps实习生 - 岗位专项评审标准

> 本文件为岗位叠加层，继承 `_base-template.md` 的通用评审框架。
> 以下内容占据总分的 **25%（岗位专项维度）**。

## 岗位专项评审维度

### 1. CI/CD 流水线 (3分)

- 流水线配置是否清晰可读（YAML 格式规范、有注释）
- 构建阶段是否合理（lint → test → build → deploy）
- 是否有缓存策略（依赖缓存、构建缓存）减少构建时间
- 部署是否有环境隔离（dev/staging/prod）
- 是否有回滚方案（蓝绿部署/金丝雀发布/版本回退）
- 敏感信息是否使用 Secrets 管理（不硬编码在配置文件中）

### 2. 容器化与编排 (3分)

- Dockerfile 是否遵循最佳实践
  - 多阶段构建（减小镜像体积）
  - 合理使用 .dockerignore
  - 不以 root 用户运行
  - 基础镜像版本是否锁定
- Docker Compose / K8s 配置是否规范
- 资源限制是否设置（CPU/Memory limits）
- 健康检查是否配置（liveness/readiness probe）
- 日志收集是否打通

### 3. 监控与告警 (3分)

- 是否配置了关键指标监控（CPU/内存/磁盘/网络）
- 应用层面是否有监控（错误率、响应时间、QPS）
- 告警规则是否合理（阈值设置、告警收敛、避免告警风暴）
- 日志是否结构化（JSON格式、有traceId/requestId）
- 是否有链路追踪（Jaeger/SkyWalking/Zipkin）

### 4. 安全与合规 (3分)

- 服务器安全基线是否达标（SSH配置、防火墙、端口最小化）
- 密钥管理是否安全（Vault/KMS，不硬编码）
- 是否有定期更新机制（系统补丁、依赖更新）
- 备份策略是否完善（频率、保留期、恢复测试）
- 网络安全（VPC/安全组/SSL证书管理）

### 5. 自动化与效率 (3分)

- 基础设施是否代码化（Terraform/Ansible/Pulumi）
- 是否有环境一键搭建能力（新人能快速拉起开发环境）
- 重复操作是否脚本化（部署脚本、数据迁移脚本）
- 是否有文档（Runbook/操作手册/故障排查指南）
- 变更是否有记录（变更日志、审批流程）

## 运维专项扣分示例（供AI参考）

| 问题 | 扣分 | 理由 |
|------|------|------|
| 密码/密钥硬编码在配置文件 | -10 | 安全零容忍 |
| Dockerfile 以 root 运行 | -5 | 安全风险 |
| 无健康检查配置 | -3 | 无法自动恢复 |
| 告警阈值设置不合理 | -3 | 告警疲劳或漏报 |
| 无备份策略 | -5 | 数据丢失风险 |
| 生产环境无监控 | -8 | 盲人开车 |
| 操作全靠手动无脚本 | -3 | 不可复现 |
