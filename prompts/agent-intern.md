# AI Agent开发实习生 - 岗位专项评审标准

> 本文件为岗位叠加层，继承 `_base-template.md` 的通用评审框架。
> 以下内容占据总分的 **25%（岗位专项维度）**。

---

## 岗位专项评审维度

### 1. Agent架构设计与实现（20分）

- **架构合理性**：
  - Agent 整体架构设计是否清晰合理
  - 模块划分是否清晰，职责是否单一
  - 各模块之间的交互是否合理
  - 是否有良好的可扩展性
- **规划能力**：
  - 是否有任务拆解和规划能力
  - 规划逻辑是否合理，能否将复杂任务拆解为可执行的子任务
  - 是否有动态调整规划的能力
- **记忆机制**：
  - 短期记忆/对话历史管理是否合理
  - 长期记忆设计是否得当（向量数据库、知识库等）
  - 记忆检索机制是否高效准确
  - 是否有记忆总结和提炼的能力
- **工具使用能力**：
  - 工具调用逻辑是否正确
  - 工具选择是否合理
  - 工具参数构造是否准确
  - 工具返回结果的解析和处理是否得当
  - 是否有工具错误处理和重试机制
- **反思与迭代能力**：
  - 是否有自我反思和纠错机制
  - 能否从错误中学习并调整策略
  - 是否有结果验证和质量检查机制

---

### 2. Prompt工程与LLM交互（20分）

- **Prompt质量**：
  - System Prompt 设计是否清晰完整
  - 是否明确定义了Agent的角色、目标、约束
  - Prompt结构是否合理，逻辑是否清晰
  - 是否有良好的指令遵循设计
- **Few-shot示例设计**：
  - 是否有合适的示例引导模型输出正确格式
  - 示例选择是否典型、有代表性
  - 示例数量是否适中
- **输出格式控制**：
  - 是否能稳定控制输出格式（JSON/Markdown等）
  - 结构化输出解析是否鲁棒
  - 是否有格式校验和纠错机制
- **上下文管理**：
  - 上下文窗口利用是否合理
  - 是否有上下文压缩/总结机制
  - 长对话场景下的信息丢失问题是否有处理
- **Token效率**：
  - 是否有Token优化意识
  - 能否在保证效果的前提下减少Token消耗
  - Prompt是否简洁高效，没有冗余信息
- **模型选择与适配**：
  - 是否根据任务特点选择合适的模型
  - 是否理解不同模型的特点和局限性
  - 能否针对不同模型调整Prompt策略

---

### 3. 工具集成与能力扩展（20分）

- **工具封装质量**：
  - 工具函数封装是否清晰，接口设计是否合理
  - 参数定义是否完整准确
  - 返回值格式是否统一规范
  - 是否有完善的错误处理
- **工具多样性**：
  - 是否能集成多种类型的工具（搜索、代码执行、API调用等）
  - 工具覆盖的场景是否全面
  - 是否有自定义工具开发能力
- **RAG与知识库集成**：
  - 知识库构建是否合理（文档切分、向量化等）
  - 检索策略是否得当（相似度检索、混合检索等）
  - 检索结果的相关性和准确性如何
  - 是否有重排序（Rerank）机制
  - 上下文注入是否自然合理
- **多模态能力**：
  - 是否能处理图片、文档等多模态输入
  - 多模态信息理解和整合能力如何
  - 多模态输出生成质量如何
- **外部API集成**：
  - API调用封装是否规范
  - 鉴权处理是否安全
  - 限流、重试、熔断等机制是否完善
  - 异常处理是否到位

---

### 4. 工程化与代码质量（20分）

- **代码结构**：
  - 项目结构是否清晰合理
  - 模块划分是否得当
  - 代码组织是否有章法
- **配置管理**：
  - 配置文件设计是否合理
  - 敏感信息（API Key等）是否通过环境变量管理
  - 是否区分开发/测试/生产环境配置
  - 配置是否有默认值和校验
- **错误处理与容错**：
  - 是否有完善的异常捕获和处理机制
  - 错误信息是否清晰，便于调试
  - 是否有降级和容错策略
  - 重试机制是否合理（指数退避等）
- **日志与监控**：
  - 日志记录是否完善
  - 是否有关键链路的追踪
  - 是否有性能监控和指标统计
  - 日志级别是否合理
- **测试覆盖**：
  - 是否有单元测试
  - 核心功能是否有测试覆盖
  - 是否有集成测试
  - 测试用例设计是否合理
- **可扩展性**：
  - 新增功能/工具是否方便
  - 是否有插件化/模块化设计
  - 接口抽象是否合理

---

### 5. 效果评估与优化（20分）

- **评估体系**：
  - 是否有明确的评估指标和方法
  - 是否建立了测试用例集（Benchmark）
  - 评估是否全面覆盖关键场景
- **效果分析**：
  - 能否准确定位效果不好的原因
  - 是否有问题诊断和分析能力
  - 能否区分是Prompt问题、模型问题还是架构问题
- **优化迭代**：
  - 是否有持续优化的意识和方法
  - 优化手段是否多样（Prompt优化、架构调整、工具增强等）
  - 优化效果是否可量化衡量
- **成本控制**：
  - 是否有Token成本意识
  - 能否在效果和成本之间取得平衡
  - 是否有成本优化的手段和实践
- **用户体验**：
  - 交互设计是否友好自然
  - 响应速度是否可接受
  - 输出质量是否符合预期
  - 是否有用户反馈收集和处理机制

## Agent专项扣分示例（供AI参考）

| 问题 | 扣分 | 理由 |
|------|------|------|
| API Key硬编码在代码中 | -10 | 严重安全隐患，零容忍 |
| Prompt设计混乱，指令不清晰 | -6 | 直接影响Agent效果，核心能力不足 |
| 没有错误处理，调用失败直接崩溃 | -5 | 鲁棒性差，工程化不足 |
| 输出格式不稳定，解析经常失败 | -6 | 影响系统可靠性，核心问题 |
| 上下文管理混乱，经常遗忘重要信息 | -7 | 严重影响对话体验和任务完成率 |
| 工具调用逻辑错误，参数构造不对 | -5 | 工具使用能力不足 |
| 没有任何测试，质量无法保障 | -4 | 工程化意识薄弱 |
| RAG检索效果差，相关度低 | -5 | 知识库利用效率低 |
| 代码结构混乱，难以维护扩展 | -4 | 工程质量差，可维护性低 |
| 没有日志和监控，问题难以排查 | -3 | 可观测性不足，运维困难 |
