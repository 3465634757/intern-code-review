# 移动端实习生 - 岗位专项评审标准

> 本文件为岗位叠加层，继承 `_base-template.md` 的通用评审框架。
> 以下内容占据总分的 **25%（岗位专项维度）**。

---

## 岗位专项评审维度

### 1. 平台规范与架构设计（20分）

**iOS 平台：**
- 是否遵循 Human Interface Guidelines (HIG) 设计规范
- 内存管理是否正确（循环引用检测、delegate 使用 weak、Block 循环引用）
- 线程安全是否考虑（UI 更新是否在主线程、多线程数据竞争）
- Info.plist 权限描述是否完整、清晰
- 是否正确使用 Autolayout / SwiftUI 布局
- ARC 内存管理是否合理，是否有内存泄漏风险

**Android 平台：**
- 是否遵循 Material Design 设计规范
- Activity/Fragment 生命周期是否正确处理
- 内存泄漏风险是否规避（Context 持有、Handler 泄漏、未注销广播/监听）
- 权限运行时申请是否正确实现（6.0+ 动态权限）
- 是否正确处理屏幕旋转、后台恢复等场景
- 组件化/模块化设计是否合理

**跨平台（Flutter/React Native）：**
- 平台特定代码是否正确隔离（Platform Channel / Native Module）
- 原生模块桥接是否安全，是否有内存泄漏风险
- 是否考虑了平台差异（字体、导航、手势、状态栏）
- 状态管理方案是否合理（Provider/Riverpod/Bloc/Redux/MobX）
- 性能是否考虑（列表复用、图片缓存、帧渲染优化）

---

### 2. 性能优化（20分）

- **列表渲染性能**：长列表是否使用懒加载/虚拟化/复用机制（RecyclerView/FlatList/ListView.builder）
- **图片处理**：是否有图片缓存策略、按需加载不同分辨率、图片压缩处理
- **启动速度**：是否有冷启动优化（懒初始化、预加载、启动任务调度）
- **帧率优化**：是否避免在主线程做重计算、动画是否流畅（60fps目标）
- **包体积优化**：是否引入过大的未使用库、资源是否压缩、是否有代码混淆
- **电池消耗**：定位、网络请求是否合理节流、后台任务是否合理调度
- **内存使用**：是否有内存泄漏、大对象处理是否合理、Bitmap 等大资源是否及时回收
- **卡顿优化**：是否避免主线程阻塞、是否使用了异步处理耗时任务

---

### 3. 数据与网络（20分）

- **离线处理**：无网络时是否有降级方案、本地缓存策略是否合理
- **数据持久化**：存储方案选择是否合理（SQLite/Realm/Core Data/SharedPreferences/UserDefaults/MMKV）
- **网络请求**：错误处理是否完善、是否有重试机制、超时设置是否合理
- **数据同步**：是否有冲突处理方案、本地与服务器数据同步策略
- **安全性**：本地敏感数据是否加密存储、API 通信是否使用 HTTPS、证书校验
- **缓存策略**：网络缓存是否合理（缓存时间、缓存大小限制、缓存清理机制）
- **数据序列化**：JSON/Model 转换是否高效、是否处理了异常数据

---

### 4. UI/UX 实现质量（20分）

- **多尺寸适配**：是否适配不同屏幕尺寸（小屏/大屏/平板/折叠屏）
- **深色模式**：是否支持 Dark Mode，颜色适配是否合理
- **手势交互**：手势操作是否自然流畅（滑动返回、下拉刷新、长按菜单）
- **动画效果**：动画是否流畅且有意义、不卡顿、不过度使用动画
- **无障碍支持**：是否支持 TalkBack/VoiceOver、大字体、高对比度
- **状态处理**：Loading/Empty/Error 状态是否都有友好的 UI 展示
- **国际化**：是否支持多语言、文案是否考虑长度差异
- **输入体验**：键盘处理是否合理、输入框是否有联想、错误提示是否友好

---

### 5. 工程化与质量保障（20分）

- **项目结构**：代码组织是否清晰、按功能/层级分包是否合理
- **状态管理**：状态管理方案是否合理、是否有状态混乱问题
- **路由管理**：导航路由是否统一管理、深链接是否支持
- **测试覆盖**：是否有单元测试、UI 测试、集成测试
- **构建配置**：是否区分开发/测试/生产环境、配置是否合理
- **代码复用**：是否有公共组件抽取、是否避免重复代码
- **日志与监控**：是否有日志系统、崩溃监控、性能监控
- **版本管理**：Git 使用是否规范、分支策略是否合理

## 移动端专项扣分示例（供AI参考）

| 问题 | 扣分 | 理由 |
|------|------|------|
| 循环引用导致内存泄漏 | -8 | 影响稳定性，可能导致崩溃 |
| 长列表未做复用/懒加载 | -5 | 严重性能问题，卡顿掉帧 |
| 主线程做网络请求/耗时计算 | -8 | 可能导致 ANR/卡顿，用户体验差 |
| 敏感数据明文存储（密码/Token） | -10 | 严重安全问题，数据泄露风险 |
| 未处理无网络状态 | -3 | 用户体验差，容易产生困惑 |
| 引入整个工具库只用一个函数 | -4 | 包体积浪费，增加安装包大小 |
| 硬编码字符串未国际化 | -3 | 可维护性差，不利于多语言扩展 |
| 图片未做缓存处理 | -4 | 浪费流量，加载速度慢 |
| 未处理生命周期导致的崩溃 | -6 | 稳定性问题，用户体验差 |
| 权限申请未说明用途 | -5 | 可能导致应用被拒，用户体验差 |
