# 移动端实习生 - 岗位专项评审标准

> 本文件为岗位叠加层，继承 `_base-template.md` 的通用评审框架。
> 以下内容占据总分的 **15%（岗位专项维度）**。

## 岗位专项评审维度

### 1. 平台规范遵循 (3分)

**iOS：**
- 是否遵循 Human Interface Guidelines
- 内存管理是否正确（循环引用、delegate 强引用）
- 线程安全（UI 更新是否在主线程）
- Info.plist 权限描述是否完整

**Android：**
- 是否遵循 Material Design 规范
- Activity/Fragment 生命周期是否正确处理
- 内存泄漏风险（Context 持有、Handler 泄漏、未注销监听）
- 权限运行时申请是否正确实现

**跨平台 (Flutter/React Native)：**
- 平台特定代码是否正确隔离
- 原生模块桥接是否安全
- 是否考虑平台差异（字体、导航、手势）

### 2. 性能优化 (3分)

- **列表渲染**：长列表是否使用懒加载/虚拟化（FlatList/RecyclerView/ListView.builder）
- **图片处理**：是否有缓存策略、按需加载不同分辨率
- **启动速度**：是否有冷启动优化（懒初始化、预加载）
- **帧率意识**：是否避免在主线程做重计算、动画是否流畅(60fps)
- **包体积**：是否引入过大的未使用的库/资源
- **电池消耗**：定位/网络请求是否合理节流

### 3. 数据与网络 (3分)

- **离线处理**：无网络时是否有降级方案、本地缓存策略
- **数据持久化**：存储方案选择是否合理（SQLite/SharedPreferences/Keychain）
- **网络请求**：错误处理是否完善、是否有重试机制
- **数据同步**：是否有冲突处理方案
- **安全性**：本地存储是否加密敏感数据、API 通信是否用 HTTPS

### 4. UI/UX 实现 (3分)

- 适配不同屏幕尺寸（小屏/大屏/平板/折叠屏）
- 深色模式是否支持
- 手势交互是否自然（滑动返回、下拉刷新、长按菜单）
- 动画是否流畅且有意义（不卡顿、不过度）
- 无障碍支持（TalkBack/VoiceOver、大字体、高对比度）

### 5. 工程实践 (3分)

- 状态管理方案是否合理（Provider/Riverpod/Bloc/Redux/MobX）
- 导航路由是否统一管理
- 代码结构是否清晰（按功能/按层级分包）
- 是否有基本的 UI 测试或集成测试
- 构建配置是否区分环境（dev/staging/prod）

## 移动端专项扣分示例（供AI参考）

| 问题 | 扣分 | 理由 |
|------|------|------|
| 循环引用导致内存泄漏 | -8 | 影响稳定性 |
| 列表未做懒加载 | -5 | 性能问题 |
| 主线程做网络请求 | -8 | ANR/卡顿 |
| 敏感数据明文存储 | -8 | 安全风险 |
| 未处理无网络状态 | -3 | 用户体验差 |
| 引入整个 lodash 用一个函数 | -3 | 包体积浪费 |
| 硬编码字符串未国际化 | -2 | 可维护性差 |
