# 后端实习生 - 岗位专项评审标准

> 本文件为岗位叠加层，继承 `_base-template.md` 的通用评审框架。
> 以下内容占据总分的 **15%（岗位专项维度）**。

## 岗位专项评审维度

### 1. API 设计 (3分)

- 是否遵循 RESTful 规范（正确的 HTTP 方法、状态码、资源命名）
- 请求/响应结构是否统一（统一的 JSON envelope、分页格式）
- 错误码和错误信息是否规范（业务错误码体系、不暴露内部细节）
- 接口文档是否同步更新（Swagger/OpenAPI 注释完整）
- 版本管理是否考虑（URL路径 vs Header 版本号）
- 请求参数校验是否在入口层完成（DTO 验证、参数边界）

### 2. 数据库操作 (3分)

- **SQL 安全**：是否存在拼接 SQL 导致注入风险
- **查询效率**：是否有 N+1 查询、全表扫描、SELECT *
- **索引使用**：WHERE/JOIN 条件字段是否有索引覆盖
- **事务处理**：事务边界是否正确、是否处理了回滚
- **ORM 使用**：是否正确使用 ORM 特性（延迟加载 vs 预加载）
- **数据一致性**：并发场景下是否考虑乐观锁/悲观锁
- **迁移脚本**：数据库变更是否有 migration 文件

### 3. 安全与鉴权 (3分)

- **输入校验**：所有外部输入是否在入口层校验（参数、Header、Body）
- **鉴权逻辑**：每个接口是否都有正确的鉴权中间件/装饰器
- **授权检查**：是否有越权漏洞（水平越权/垂直越权）
- **密码处理**：是否使用 bcrypt/argon2 哈希（禁止 MD5/SHA1）
- **日志脱敏**：日志中是否包含密码、Token、身份证号等敏感信息
- **CORS 配置**：是否过于宽松（`Access-Control-Allow-Origin: *`）
- **依赖安全**：是否引入有已知漏洞的包

### 4. 并发与性能 (3分)

- **阻塞操作**：是否在请求处理中执行了同步IO/耗时操作
- **缓存策略**：热点数据是否有缓存、缓存是否有一致性方案
- **异步处理**：耗时任务是否放到消息队列/后台任务
- **连接池**：数据库/Redis 连接池配置是否合理
- **限流熔断**：对外部依赖的调用是否有超时、重试、熔断
- **资源释放**：文件句柄、数据库连接是否正确关闭（defer/finally）

### 5. 测试覆盖 (3分)

- 核心业务逻辑是否有单元测试（Service 层关键方法）
- 边界条件是否覆盖（空值、极大值、并发场景）
- 集成测试是否覆盖关键接口
- 测试是否独立（不依赖外部环境、可重复运行）
- Mock 是否合理（不过度 Mock 导致测试失去意义）

## 后端专项扣分示例（供AI参考）

| 问题 | 扣分 | 理由 |
|------|------|------|
| SQL 字符串拼接 | -10 | 注入风险，零容忍 |
| 日志中打印用户密码 | -10 | 数据泄露 |
| 接口无鉴权 | -8 | 未授权访问 |
| SELECT * 查询 | -3 | 性能隐患 |
| 事务未回滚 | -5 | 数据一致性风险 |
| 密码用 MD5 哈希 | -8 | 不安全的哈希算法 |
| 无任何测试 | -5 | 代码质量无法保证 |
| 同步调用外部API | -3 | 阻塞请求线程 |
