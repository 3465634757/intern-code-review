 # 后端实习生 - 岗位专项评审标准

> 本文件为岗位叠加层，继承 `_base-template.md` 的通用评审框架。
> 以下内容占据总分的 **25%（岗位专项维度）**。

---

## 岗位专项评审维度

### 1. API 设计与规范（20分）

- 是否遵循 RESTful 规范（正确的 HTTP 方法、状态码、资源命名）
- 请求/响应结构是否统一（统一的 JSON envelope、分页格式、错误码体系）
- 错误码和错误信息是否规范（业务错误码体系、不暴露内部细节）
- 接口文档是否同步更新（Swagger/OpenAPI 注释完整、参数说明清晰）
- 版本管理是否考虑（URL路径版本 vs Header版本）
- 请求参数校验是否在入口层完成（DTO 验证、参数边界检查、类型校验）
- 接口幂等性是否考虑（重复请求处理、幂等键设计）

---

### 2. 数据库设计与操作（20分）

- **SQL 安全**：是否存在拼接 SQL 导致注入风险，是否使用参数化查询
- **查询效率**：是否有 N+1 查询问题、全表扫描、SELECT * 等低效写法
- **索引使用**：WHERE/JOIN 条件字段是否有索引覆盖，是否合理使用联合索引
- **事务处理**：事务边界是否正确、是否处理了回滚、事务隔离级别是否合理
- **ORM 使用**：是否正确使用 ORM 特性（延迟加载 vs 预加载、关联查询）
- **数据一致性**：并发场景下是否考虑了乐观锁/悲观锁
- **数据库迁移**：表结构变更是否有 migration 脚本，是否考虑了数据迁移
- **大数据量处理**：是否考虑分批处理、游标查询，避免内存溢出

---

### 3. 安全与鉴权（20分）

- **输入校验**：所有外部输入是否在入口层校验（参数、Header、Body、Cookie）
- **鉴权逻辑**：每个接口是否都有正确的鉴权中间件/装饰器，是否存在未授权访问
- **授权检查**：是否有越权漏洞（水平越权/垂直越权），是否正确校验数据归属
- **密码处理**：是否使用 bcrypt/argon2 等安全哈希算法（禁止 MD5/SHA1 明文）
- **日志脱敏**：日志中是否包含密码、Token、身份证号等敏感信息
- **CORS 配置**：是否过于宽松（Access-Control-Allow-Origin: *），是否正确配置
- **依赖安全**：是否引入了已知有漏洞的包版本
- **文件上传**：是否有文件类型校验、大小限制、路径遍历防护

---

### 4. 并发与性能优化（20分）

- **阻塞操作**：是否在请求处理中执行了同步IO、耗时计算等阻塞操作
- **缓存策略**：热点数据是否有缓存、缓存是否有一致性方案、缓存穿透/击穿/雪崩防护
- **异步处理**：耗时任务是否放到消息队列/后台任务中处理
- **连接池**：数据库/Redis/HTTP 连接池配置是否合理
- **限流熔断**：对外部依赖的调用是否有超时、重试、熔断机制
- **资源释放**：文件句柄、数据库连接、网络连接是否正确关闭（defer/finally/try-with-resources）
- **内存使用**：是否有内存泄漏风险，大对象处理是否合理

---

### 5. 测试与质量保障（20分）

- 核心业务逻辑是否有单元测试（Service 层关键方法）
- 边界条件是否覆盖（空值、极值、异常场景、并发场景）
- 集成测试是否覆盖关键接口（API 层端到端测试）
- 测试是否独立（不依赖外部环境、可重复运行、测试数据隔离）
- Mock 是否合理（不过度 Mock 导致测试失去意义，正确 Mock 外部依赖）
- 测试数据构造是否合理（工厂模式、fixture，避免硬编码）
- 是否有性能测试或压力测试意识

## 后端专项扣分示例（供AI参考）

| 问题 | 扣分 | 理由 |
|------|------|------|
| SQL 字符串拼接导致注入风险 | -10 | 严重安全漏洞，零容忍 |
| 日志中打印用户密码/Token | -10 | 数据泄露，严重安全问题 |
| 接口完全没有鉴权 | -8 | 未授权访问，严重安全问题 |
| SELECT * 全表查询 | -3 | 性能隐患，浪费资源 |
| 事务边界错误或未回滚 | -5 | 数据一致性风险 |
| 密码使用 MD5/SHA1 哈希 | -8 | 不安全的哈希算法，易被破解 |
| 完全没有任何测试 | -5 | 代码质量无法保障 |
| 同步调用外部API无超时 | -4 | 可能导致线程阻塞，服务雪崩 |
| N+1 查询问题 | -4 | 严重性能问题，数据库压力大 |
| 硬编码数据库连接信息 | -10 | 敏感信息泄露，零容忍 |

## 后端专项加分项

- ✅ 主动进行性能优化（SQL优化、缓存策略、异步处理）
- ✅ 完善的单元测试和集成测试，测试覆盖率高
- ✅ 有完整的错误处理和异常捕获机制
- ✅ 接口设计清晰，文档完善，易于使用
- ✅ 有监控、埋点、告警意识
- ✅ 代码注释清晰，关键逻辑有说明
- ✅ 考虑了系统的可扩展性和可维护性
